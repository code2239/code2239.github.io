## 2. 底层技术栈

DeepSeek 完全自研基础大模型，**不调用第三方大模型 API**。其核心技术栈如下：

### 模型架构
- **MoE（混合专家）架构**：V4-Pro 每层 384 个专家，每次推理激活 6 个（49B 活跃参数 / 1.6T 总参数）；V4-Flash 激活 13B / 284B 总参数
- **DSA2 混合注意力机制**：整合 CSA（压缩稀疏注意力）+ HCA（重度压缩注意力），替代 V3 的 Multi-head Latent Attention，实现 90% KV 缓存缩减
- **mHC（流形约束超连接）**：替代标准残差连接，提升深层网络训练稳定性
- **Muon 优化器**：加速收敛
- **多 Token 预测（MTP）**：延续 V3 的推理加速策略
- **FP4 + FP8 混合精度**：显存需求在 FP8 基础上再降一半

### 训练基础设施
- **华为昇腾 910B/950PR 芯片**：V4 是首个全面适配华为 CANN 架构的 DeepSeek 旗舰模型
- V4-Flash 预训练 32T tokens，V4-Pro 预训练 33T tokens

### 推理模式
- **Non-Think**：单次直出，无思维链
- **Think-High**：标准推理链
- **Think-Max**：最大推理深度 + 自我纠错

## 4. 功能全景解析

### 主要功能
以极低成本提供接近闭源顶流的大模型服务，面向开发者、中小企业、以及有国产化/私有化部署需求的企业用户。

### 核心功能与核心技术

#### ① 百万 Token 上下文处理（核心技术：DSA2 稀疏注意力）
- 1M token 上下文窗口，384K token 最大输出（业内最长，3 倍于竞品）
- KV 缓存仅为 V3.2 的 10%，推理 FLOPs 仅为 27%
- **适用场景**：一次性加载整本技术手册、跨国并购合同、完整代码库

#### ② 多步 Agent 自主编程（核心技术：Think-Max 推理 + 工具调用）
- SWE-bench Verified 80.6%，Codeforces 3206 分
- 支持 30+ 步连续工具调用（浏览器、终端、文件系统、API）
- 384K 输出能力支持单轮多文件重构
- **适用场景**：全栈项目开发、代码审查、自动化测试、CI/CD 脚本

#### ③ FP4 精度 + 消费级部署（核心技术：INT4 量化 + 华为昇腾适配）
- V4-Flash 经 INT4 量化可在 RTX 4090（24GB 显存）上运行
- 支持 llama.cpp 跨平台（Mac / Windows / 树莓派）
- 已适配华为昇腾 910B/950PR，实现国产算力闭环
- **适用场景**：金融终端本地推理、工业控制离线 AI、隐私敏感场景

## 附录：V4 模型差异化使用指南

| 场景 | 推荐模型 | 原因 |
|---|---|---|
| 日常写代码 / Debug / CRUD | **Flash** | 差距微乎其微，成本低 10 倍 |
| 复杂算法 / Agent 多步编程 | **Pro** | Codeforces 3206 vs 3052，差距决定性的 |
| 普通报告 / 周报 / 翻译 | **Flash** | 质量感知不到差异 |
| 行业研究 / 法律 / 医疗 | **Pro** | 事实准确性领先 23.8 分 |
| 画图表（生成代码） | **Flash**（简单）/ **Pro**（复杂可视化） | 模板化用 Flash，D3.js 等用 Pro |
| 做 PPT | **Flash** 为主 | 结构化任务，Flash 足够 |
| 全栈项目开发 | **Flash** 日常 + **Pro** 救急 | 路由架构：95% Flash，5% Pro |
| 客服 / RAG / 摘要 / 闲聊 | **Flash** | 成本效率碾压 |
| 私有化本地部署 | **Flash**（RTX 4090 可跑） | Pro 需要集群 |

> ⚠️ **注意**：DeepSeek V4 目前不支持多模态（图片/视频/音频），视觉版仍在规划中。需要多模态时建议使用 Gemini 3.1 Pro 或 GPT-5.5。
