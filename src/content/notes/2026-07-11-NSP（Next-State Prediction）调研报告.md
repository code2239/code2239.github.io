## 2. 底层技术解析

### 2.1 什么是 NSP？

> **NSP（Next-State Prediction）** 是一种全新的 AI 学习范式——模型不再预测"下一个词是什么"，而是预测"世界的下一个物理状态是什么"。

### 2.2 三代范式的本质区别

```
NTP（Next-Token Prediction）         GPT 系列、LLaMA
   │  预测：下一个词
   │  理解：语言统计规律
   │  盲区：不懂物理世界
   ▼

NPF（Next-Pixel/Frame Prediction）    Sora、视频模型
   │  预测：下一帧像素
   │  理解：视觉表面连续性
   │  盲区：像素变化≠物理理解
   ▼

NSP（Next-State Prediction）⭐⭐⭐    智源 Emu3.5、悟界·Orca
   │  预测：下一个物理状态
   │  理解：物理因果规律
   │  突破：从"看起来像"到"真的懂"
```

### 2.3 技术实现路径

NSP 的核心是一套**统一物理潜空间表征 + 状态转移建模**框架：

```
多模态输入（文本/图像/视频/动作）
         │
         ▼  离散化为统一 Token 序列
┌─────────────────────────────────┐
│       统一 Token 化空间          │
│  "杯子" = token_id_1423         │
│  杯子图像 = [token_id_1423, ...]│
│  杯子掉落视频 = [token_seq]     │
└──────────┬──────────────────────┘
           │  Self-Attention + State Transition
           ▼
┌─────────────────────────────────┐
│    状态转移学习（State Dynamics） │
│  S₁（杯在桌沿）                 │
│    → 重力作用                   │
│  S₂（杯开始倾斜）               │
│    → 力矩计算                   │
│  S₃（杯脱离桌面）               │
│    → 自由落体                   │
└──────────┬──────────────────────┘
           ▼
┌─────────────────────────────────┐
│    输出：下一个物理状态预测       │
│  文本描述 / 视频帧 / 动作指令    │
└─────────────────────────────────┘
```

### 2.4 智源悟界·Emu3.5 技术架构

| 维度 | 规格 |
|------|------|
| **参数量** | **34B（340 亿）** |
| **训练 Token** | **超 10 万亿**（两阶段预训练约 13 万亿）|
| **视频数据** | 6,300 万条长视频，总时长 **790 年**（仅占全网公开视频不到 1%）|
| **数据预处理** | Whisper ASR + PySceneDetect 关键帧提取 + 图文交错序列 |
| **训练流程** | 两阶段预训练 → SFT（1,500 亿样本）→ 大规模强化学习 → DiDA 快速适配 |

### 2.5 三大核心技术创新

#### 1️⃣ 大规模多模态预训练
- 文本、图像、视频、动作全部离散化为**统一 Token 序列**
- 模型不再区分模态——"一张图"和"一句话"用同一套"世界积木"表示
- 两阶段训练：基础预训练（10万亿 Token）→ 高分辨率/高质量微调

#### 2️⃣ 大规模原生多模态强化学习（RL）
- 多模态领域**首次**在自回归架构下落地大规模 RL
- 构建综合奖励系统：通用性 + 任务特异性 + 统一性
- 多维奖励避免单目标过拟合

#### 3️⃣ DiDA（离散扩散自适应）推理加速
- 解决自回归架构图像生成慢的瓶颈
- 将离散扩散公式扩展到视觉 Token
- **单图推理速度提升近 20 倍**，媲美顶尖闭源扩散模型

## 4. 功能全景解析

### 核心能力（一句话概括）
> NSP 让 AI 从"续写文字"升级为"理解世界因果"——输入多模态信息，输出对下一个物理状态的预测与推理。

### 4.1 🧠 物理世界理解

| 维度 | 说明 |
|------|------|
| **物体理解** | 识别物体身份、位置、姿态、材质属性 |
| **关系推理** | 理解物体间的空间关系、因果关系 |
| **物理规律** | 内化重力、摩擦、碰撞、流体等基本物理规律 |

**王仲远经典举例**：
> 一瓶未开封的水和盛满咖啡的杯子放在桌旁。人类能预测：水瓶跌落只有撞击声，咖啡杯会洒落一地。这就是世界模型。传统 AI 缺乏这种"下一个物理状态"的预测能力。

### 4.2 🔮 下一个状态预测

| 输入 | 输出 | 示例 |
|------|------|------|
| 文本描述 | 物理状态描述 | "杯子在桌沿"→"即将掉落" |
| 图像 | 下一时刻图像 | 静态场景→预测物体运动趋势 |
| 视频 | 延续视频 | 前 5 秒→预测后 5 秒的物理演变 |
| 动作指令 | 动作结果 | "把杯子拿起来"→预测手的位置变化 |

### 4.3 🎯 意图到规划

- 理解高层级人类意图（如"如何做咖啡拉花""如何叠衣服"）
- 自主生成详细、连贯的多步骤行动路径
- 在统一框架内无缝融合理解、规划与模拟

### 4.4 🔄 动态世界模拟

- 预测物理动态、时空演化和长时程因果关系
- 支持反事实推理（Counterfactual Reasoning）："如果不这样做会怎样？"
- 为具身操控提供认知基础

### 4.5 🏗️ 第三个 Scaling 范式

王仲远提出：NSP 开启了继"语言预训练 Scaling"和"后训练/推理 Scaling"之后的**第三个 Scaling 范式**：

| Scaling 范式 | 代表 | 核心特征 |
|:---:|------|---------|
| **第一代** | GPT 系列 | 语言预训练参数规模扩展 |
| **第二代** | o1、DeepSeek-R1 | 后训练 + 推理时间扩展 |
| **第三代（NSP）** | **智源 Emu3.5** | **多模态大一统 + 大规模 RL + 基础设施复用** |

## 5. 功能实现案例

### 案例：悟界·RoboBrain Orca —— NSP 驱动的具身大脑

**场景**：机器人需要完成"从桌上拿起一个易碎的高脚杯，放到洗碗机上层支架"的任务。

**传统 VLA 方案**：
```
输入：视觉画面 → 直接输出机械臂关节角度
问题：每种杯子、每种摆放位置都需要大量真实数据训练
      遇到没见过的杯子或布局，泛化能力差
```

**NSP 方案（悟界·Orca）**：
```
输入：7 路视觉画面 + 机械臂状态
    ↓
Step 1：物理状态抽象
   └─ 将场景抽象为统一潜空间表征：
      高脚杯（易碎材质，重心偏高，杯柄脆弱）
      桌面高度 0.75m，洗碗机上层高度 0.4m
      当前抓取角度 30°
    ↓
Step 2：NSP 状态推演
   └─ 在潜空间中预演多条路径的物理结果：
      ├─ 路径 A：直接抓杯口 → 预测→ 杯柄应力超标 → 可能断裂 ❌
      ├─ 路径 B：抓杯柄底部 → 预测→ 应力安全 → 继续推演
      │              ↓ 放入洗碗机 → 预测→ 高度冲突 → 旋转 15° ✅
      └─ 路径 C：先平移再抓取 → 预测→ 耗时较长但稳定 → 备选
    ↓
Step 3：选择最优路径 → 输出动作序列
```

**结果**：NSP 让机器人在**没有真实数据的情况下**，通过在潜空间中"预演"物理结果来选择最优动作，泛化到从未见过的杯型和布局。

## 6. 主要竞品/对比范式

### 6.1 三代范式对比

| # | 范式 | 代表 | 核心逻辑 | 根本局限 |
|---|------|------|---------|---------|
| 1 | **NTP（Next-Token Prediction）** | GPT-5、LLaMA 4 | 预测下一个词，学习语言统计规律 | **无物理理解**，不懂因果 |

## 7. 横向比较

### 7.1 三代范式多维对比

| 对比维度 | **NTP（Next-Token）** | **NPF（Next-Frame）** | **NSP（Next-State）** |
|:---|:---|:---|:---|
| **预测单位** | 单个词/子词 | 整帧像素阵列 | **结构化状态向量**（位置/速度/力/关系）|
| **训练数据** | 文本语料 | 视频帧序列 | **多模态序列**（视频/传感/物理仿真）|
| **学习目标** | 语言概率分布 | 像素级视觉连续性 | **状态转移概率分布** |
| **核心能力** | 语言流畅性 | 视觉时间连贯性 | **因果物理理解** |
| **物理理解** | ❌ 完全不懂 | ⚠️ 隐式、不可靠 | ✅ **显式因果建模** |
| **可否规划** | ❌ 不能 | ⚠️ 有限 | ✅ **可预演多路径** |
| **算力需求** | 中 | **极高**（像素级生成）| 高（但潜空间远低于像素）|
| **长程一致性** | ✅ 文本可很长 | ❌ 误差指数累积 | ⚠️ 潜空间好于像素但仍有挑战 |
| **代表性基准** | MMLU、HumanEval | VBench、Physics-IQ | **WorldModelBench** |
| **最适用场景** | 对话、写作、编程 | 视频生成、创意内容 | **机器人、自动驾驶、科学模拟** |

### 7.2 各范式最适用场景

| 范式 | 最佳场景 | 不应使用的场景 |
|------|---------|--------------|
| **NTP** | 聊天、翻译、文档编写、代码生成 | 需要物理理解的机器人/自动驾驶 |
| **NPF** | 视频创作、游戏美术、数据增强 | 需要精确物理推理的生产级系统 |
| **NSP** | **机器人操控、自动驾驶规划、科学模拟** | 纯文本对话（大材小用）|

## 附：注意区分——NSP 的两种含义

在学术文献中，"NSP"存在**两个不同概念**，注意不要混淆：

| | **Next-State Prediction（本报告主题）** | **Next-Scale Prediction** |
|---|---|---|
| **领域** | 世界模型 / AGI / 具身智能 | 计算机视觉 / 高效生成 |
| **核心思想** | 预测世界下一个物理状态 | 从粗到细预测图像尺度 |
| **提出者** | **智源研究院（王仲远）** | VAR 论文（Tian et al.）|
| **代表作** | **Emu3.5、悟界·Orca** | VAR（Visual AutoRegressive）|
| **效果** | 因果物理理解 | 比扩散模型快 20× |
| **⚡ 区分关键** | 关注"世界状态怎么变" | 关注"图像怎么从模糊变清晰" |
