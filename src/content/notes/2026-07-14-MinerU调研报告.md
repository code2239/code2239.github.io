## 2. 底层技术栈

> MinerU 是 **AI 赋能的应用软件**，底层依赖多个 AI 模型和深度学习框架。

### 2.1 调用的基础大模型

| 模型 | 用途 | 说明 |
|------|------|------|
| **MinerU2.5-Pro-2604-1.2B** | **VLM 主模型**（v3.x 默认） | 自研多模态模型，1.2B 参数，性能超越 Gemini 2.5 Pro 和 GPT-4o |
| **detectron2** | 版面检测（旧版） | Facebook AI 的检测框架，用于识别标题/段落/图片/表格位置 |
| **PaddleOCR** | 文字识别 | 百度 OCR 模型，支持 109 种语言 |
| **UniMERNet** | 公式识别 | 将数学公式转为 LaTeX 格式 |
| **StructTable** | 表格识别 | 识别并还原表格结构为 HTML 格式 |
| **layoutreader** | 阅读顺序排序 | 判断文档内容的正确阅读顺序 |
| **doclayout_yolo** | 版面元素检测 | YOLO 系列模型，检测文档中的各类元素 |

### 2.2 底层深度学习框架

- **PyTorch**：所有模型的核心运行引擎（就是你在安装的那一个）
- **ONNX Runtime**：部分模型推理优化
- **vLLM / sglang**：大模型推理加速引擎

### 2.3 技术架构图（简化）

```
输入：PDF / DOCX / PPTX / XLSX / 图片
              ↓
      ┌──────────────┐
      │   预处理层     │  ← PyMuPDF / pdfplumber 解析原始文件
      └──────────────┘
              ↓
      ┌──────────────┐
      │   版面分析     │  ← detectron2 / doclayout_yolo 检测区域
      │  (Layout)     │     识别：标题、正文、表格、图片、公式、页眉页脚
      └──────────────┘
              ↓
      ┌──────────────┐
      │   内容识别     │  ← OCR (PaddleOCR) / 公式识别 / 表格识别
      │  (Recognition)│     把每个区域的内容转成文字
      └──────────────┘
              ↓
      ┌──────────────┐
      │   后处理 &     │  ← 阅读顺序排序、跨页合并、Markdown 组装
      │   Markdown 生成│
      └──────────────┘
              ↓
输出：Markdown / JSON
```

### 2.4 是否支持用户自己使用 API 调用大模型

**支持**。MinerU v3.x 提供灵活的模型配置选项：

- **内置模型**：默认使用自研的 MinerU2.5-Pro 模型
- **替换 VLM 模型**：可以切换为其他 VLM（视觉语言模型）作为解析后端
- **纯 pipeline 模式**：可以用传统的版面检测 + OCR 流水线，不依赖大模型
- **API 模式**：`mineru-open-sdk` 支持远程调用 MinerU 云端服务

## 4. 功能全景解析

### 4.1 主要功能（一句话概括）

> MinerU 是一个智能文档解析引擎，能将 PDF、Word、PPT、Excel、图片等文件转换为结构化的 Markdown 和 JSON，为 LLM、RAG、Agent 提供"AI-Ready"的数据输入。

### 4.2 核心功能与核心技术

#### 4.2.1 多格式输入支持（v3.x）
| 格式 | 支持情况 | 说明 |
|------|---------|------|
| **PDF** | ✅ **原生支持** | 扫描件、电子档均支持 |
| **DOCX** | ✅ **原生解析**（v3.0+） | 无需转 PDF，速度快 10-100 倍 |
| **PPTX** | ✅ **原生解析**（v3.1+） | 直接解析幻灯片 |
| **XLSX** | ✅ **原生解析**（v3.1+） | 直接解析电子表格 |
| **图片** | ✅ | PNG/JPG/WebP 等常见格式 |

**关键技术**：传统工具需要将 Office 文件转 PDF → 图片 → OCR 三步走，MinerU v3.x **直接原生读取文件结构**，速度提升 10-100 倍，且几乎不需要 GPU。

#### 4.2.2 版面分析（Layout Analysis）
- **功能**：自动检测文档中的各种区域类型
- **识别类型**：标题、正文段落、图片、表格、公式、页眉、页脚、脚注、页码
- **关键技术**：基于 detectron2 / doclayout_yolo 的深度学习版面检测模型
- **精度**：OmniDocBench 评分 **86.2**（pipeline 模式）

**参考**：[版面分析技术](https://github.com/opendatalab/MinerU)

#### 4.2.3 OCR 文字识别
- **功能**：识别图片中的文字
- **语言支持**：**109 种语言**
- **关键技术**：PaddleOCR（百度 OCR 引擎），PP-OCRv5
- **中英混合**：自动识别中英文混合排版

#### 4.2.4 公式识别（LaTeX）
- **功能**：将数学公式转换为 LaTeX 代码
- **关键技术**：自研 UniMERNet 模型
- **支持**：行内公式、行间公式、复杂矩阵、微积分符号

#### 4.2.5 表格识别
- **功能**：识别表格结构并还原为 HTML/Markdown 格式
- **关键技术**：StructTable + RapidTable（10x 加速）
- **跨页表格**：v3.x 支持跨页表格合并

#### 4.2.6 VLM 模式（v2.5+）
- **功能**：使用多模态大模型直接理解文档内容
- **模型**：MinerU2.5-Pro-2604-1.2B（1.2B 参数）
- **性能**：OmniDocBench 评分 **90.67**，超越 Gemini 2.5 Pro 和 GPT-4o
- **速度**：2.12 页/秒（A100 GPU）

**参考**：[MinerU2.5 模型评测](https://neurohive.io/en/state-of-the-art/mineru2-5-open-source-1-2b-model-for-pdf-parsing-outperforms-gemini-2-5-pro-on-benchmarks/)

#### 4.2.7 超长文档支持（v3.0+）
- **功能**：处理数千页的超长文档
- **关键技术**：**滑窗机制（Sliding Window）** + 流式写入磁盘
- **能力**：8GB 内存可稳定处理 **10,000+ 页**文档
- **对比**：之前需要 128GB+ 内存才能处理 ~3,000 页

#### 4.2.8 多 GPU 高并发（v3.0+）
- **功能**：多卡并行处理
- **关键技术**：**mineru-router** 编排层——自动任务路由、负载均衡、多线程并发
- **参考**：[MinerU v3.0 更新](https://cloud.tencent.com.cn/developer/article/2663052)

## 5. 功能实现案例

### 案例 1：学术论文 PDF 转 Markdown

**场景**：下载一篇 arxiv 论文，用 MinerU 转为 Markdown，方便阅读和检索。

```powershell
# 用你正在安装的 MinerU
conda activate MinerU

# 下载一篇论文
curl -O https://arxiv.org/pdf/2303.08774.pdf

# 解析为 Markdown
mineru -p "2303.08774.pdf" -o ./output
```

**结果**：
```
output/
├── 2303.08774.md              ← 完整的 Markdown 文件
│   ├ ✅ 标题层级保留（# ## ###）
│   ├ ✅ 数学公式自动转为 LaTeX（$E=mc^2$）
│   ├ ✅ 表格转为 Markdown 表格
│   ├ ✅ 图片被提取到 images/ 目录
│   └ ✅ 引用格式保留
├── images/                    ← 提取的图片文件
└── 2303.08774.json            ← 结构化 JSON（可用于 RAG 系统）
```

**技术要点**：整个过程涉及版面分析 → OCR → 公式识别 → 表格识别 → 阅读顺序排序 → Markdown 组装，全部在后台自动完成。

### 案例 2：RAG 知识库文档预处理

**场景**：公司有 50 份 PDF 合同、100 份 Word 文档，需要导入 RAG 系统做问答。

```powershell
# 批量处理整个目录
mineru -p D:\documents\ -o D:\rag_ready\

# 每个文件都转为结构化 JSON，可以直接喂给 LangChain / LlamaIndex
```

**结果**：非结构化文档 → AI-Ready 的结构化数据，可以直接导入向量数据库。

### 案例 3：扫描版 PDF 发票识别

**场景**：收到的发票是扫描件（图片型 PDF），需要提取关键信息。

```powershell
# MinerU 自动识别这是扫描件，自动启用 OCR
# 109 种语言自动检测，不需要手动指定语言
mineru -p "invoice.pdf" -o ./output
```

## 一句话锐评

> **MinerU 不仅仅是一个 PDF 解析工具，它是 AI 时代的"文档翻译官"——把人类世界积累了几十年的 PDF、Word、PPT 等非结构化文档，翻译成 AI 能理解的结构化数据。开源免费、中文友好、支持国产芯片、性能超越 GPT-4o，这些标签组合在一起，让它成为 RAG/Agent 时代的标配基础设施。你选 MinerU 来解析 PDF，是一个很聪明且与时俱进的选择。**

### 对你来说意味着什么

你在 Windows 上安装 MinerU，本质上是在装一套完整的**文档解析生产线**：

```
你的 PDF 文档
    ↓
MinerU（你正在安装的）
    ├── PyTorch（你换了 CUDA 版）
    ├── detectron2（版面检测）
    ├── PaddleOCR（文字识别）
    └── 各 AI 模型（公式、表格识别）
    ↓
结构化的 Markdown / JSON
```

装好之后，你就能：
- `mineru -p "论文.pdf" -o ./output` — 解析单篇论文
- `mineru -p D:\所有文档\ -o D:\解析结果\` — 批量处理整个文件夹
- `mineru gradio` — 启动网页界面，拖拽上传
