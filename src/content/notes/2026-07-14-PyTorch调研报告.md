## 2. 底层技术栈

### 2.1 基础大模型

> PyTorch 是**深度学习框架**，不是 AI 应用本身。它不调用任何基础大模型，而是**用来构建和训练基础大模型的工具**。

```
GPT-4、Claude、Llama、Stable Diffusion……
         ↑
    用的是 PyTorch 等框架来训练的
         ↑
PyTorch 本身不是 AI 模型，是造模型的"机床"
```

### 2.2 核心技术架构

#### 张量计算引擎（Tensor Computation）
- **核心数据类型**：`torch.Tensor` — 类似 NumPy 数组，但支持 GPU 加速
- **自动求导（Autograd）**：自动计算所有运算的梯度，是训练神经网络的基础

```python
# PyTorch 的核心——自动求导
import torch
x = torch.tensor([1.0, 2.0, 3.0], requires_grad=True)
y = x ** 2 + 3 * x
y.sum().backward()        # 自动计算导数
print(x.grad)             # 输出: tensor([5., 7., 9.])
```

#### 动态计算图（Define-by-Run）
PyTorch 最核心的差异化技术——计算图在**运行过程中动态构建**，而不是预先定义好：

| 框架 | 计算图 | 特点 |
|------|--------|------|
| **PyTorch** | **动态图**（Eager Execution） | 想怎么 debug 就怎么 debug，写起来像普通 Python |
| TensorFlow（旧版） | 静态图（Define-and-Run） | 先建图再执行，debug 困难 |

```python
# PyTorch 的动态图——随心所欲，像写普通代码
for i in range(10):
    if i % 2 == 0:
        x = x + 1
    else:
        x = x * 2
    # 每次循环的计算图都是动态构建的
```

#### torch.compile（2.0+ 核心特性）
- **原理**：通过 TorchDynamo 捕获 Python 代码执行轨迹，将其编译为高效的 GPU 内核
- **效果**：一行代码提升 30-60% 训练速度
- **底层技术**：TorchDynamo（捕获图） + TorchInductor（生成 GPU 代码）

```python
model = torch.compile(model)  # 一行代码加速
```

#### CUDA 加速
- PyTorch 支持通过 CUDA 调用 NVIDIA GPU
- 同样的代码，在 GPU 上比 CPU 快 **10-50 倍**

```python
device = torch.device("cuda" if torch.cuda.is_available() else "cpu")
model.to(device)  # 把模型搬到 GPU 上
```

#### 分布式训练栈
| 组件 | 功能 |
|------|------|
| **FSDP** (Fully Sharded Data Parallel) | ZeRO-3 实现，支持大模型训练 |
| **DTensor** | 分布式张量，自动切分到多卡 |
| **torch.distributed** | 底层分布式通信（NCCL/Gloo） |
| **TorchRL** | 强化学习库 |

## 4. 功能全景解析

### 4.1 主要功能（一句话概括）

> PyTorch 是一个端到端的深度学习框架，提供从张量计算、自动求导到分布式训练和模型部署的全链路能力，是当前 AI 研究和开发的主流平台。

### 4.2 核心功能与核心技术

#### 4.2.1 张量计算（Tensor Computation）
- **核心**：`torch.Tensor` — GPU 加速的多维数组
- **关键技术**：零成本内存视图（views）、广播语义（broadcasting）、惰性求值（lazy evaluation）

```python
x = torch.randn(3, 224, 224)  # 一张图片：3通道，224x224
y = x.to("cuda")              # 搬到 GPU 上，所有后续计算自动在 GPU 执行
```

#### 4.2.2 自动求导（Autograd）
- **核心**：自动计算任意计算图的梯度
- **关键技术**：反向模式自动微分（reverse-mode AD），通过记录运算历史构建计算图

**参考**：[Autograd 文档](https://pytorch.org/docs/stable/autograd.html)

#### 4.2.3 神经网络模块（torch.nn）
- **核心**：提供所有标准的神经网络层、损失函数、优化器
- **关键技术**：`nn.Module` 作为所有模型的基类，提供参数管理和状态序列化

```python
class MyModel(nn.Module):
    def __init__(self):
        super().__init__()
        self.fc = nn.Linear(784, 10)
    def forward(self, x):
        return self.fc(x)
```

**参考**：[torch.nn 文档](https://pytorch.org/docs/stable/nn.html)

#### 4.2.4 即时编译（torch.compile）
- **核心**：一行代码将 PyTorch 代码编译为高效的 GPU 内核
- **关键技术**：TorchDynamo（捕获图）+ TorchInductor（生成 Triton/CUDA 代码）
- **效果**：训练速度提升 30-60%，推理速度提升 2-4 倍

**参考**：[torch.compile 介绍](https://pytorch.org/tutorials/intermediate/torch_compile_tutorial.html)

#### 4.2.5 分布式训练
- **核心**：支持从单机多卡到多机多卡的大规模训练
- **关键技术**：FSDP（ZeRO-3）、DTensor、NCCL 通信库
- **能力**：可训练千亿参数级别的模型

**参考**：[分布式训练文档](https://pytorch.org/docs/stable/distributed.html)

#### 4.2.6 模型部署（TorchScript + ExecuTorch）
- **TorchScript**：将 PyTorch 模型序列化为可跨平台运行的文件格式
- **ExecuTorch**（2024+）：轻量级 C++ 运行时，支持在手机、嵌入式设备上运行

**参考**：[ExecuTorch 文档](https://pytorch.org/executorch/)

#### 4.2.7 生态系统成员

## 5. 功能实现案例

### 案例 1：用预训练模型做图像分类（5 行代码）

**场景**：给一张图片分类，判断它是什么物体。

```python
import torch
from torchvision import models, transforms
from PIL import Image

# 1. 加载预训练好的 ResNet-50 模型
model = models.resnet50(pretrained=True)
model.eval()

# 2. 加载并预处理图片
img = Image.open("cat.jpg")
transform = transforms.Compose([
    transforms.Resize(256),
    transforms.CenterCrop(224),
    transforms.ToTensor(),
])
input_tensor = transform(img).unsqueeze(0)

# 3. 推理
with torch.no_grad():
    output = model(input_tensor)

# 4. 输出结果
_, predicted = output.max(1)
print(f"类别编号: {predicted.item()}")
# → 类别编号: 285（对应 "Egyptian cat"）
```

**关键点**：
- `torchvision` 提供的预训练模型已经在大规模数据集上训练好了，开箱即用
- `torch.no_grad()` 告诉 PyTorch "不需要计算梯度"，推理更快
- 所有操作在 GPU 上自动加速

## 一句话锐评

> **PyTorch 是深度学习领域的"普通话"——不是功能最花哨的，也不是速度最快的，但它是目前**生态最大、社区最活跃、上手最舒服**的框架。无论你是造模型（AI 研究员）还是用模型（像你用 MinerU 跑 PDF 解析），PyTorch 都是最值得投资的技能栈。对于你当下的场景（装 MinerU），它只是在后台默默帮你跑 GPU 计算的引擎，装好 CUDA 版就对了。**

---

### 对你来说 PyTorch 意味着什么

你现在用 PyTorch 的方式和我刚才写的"开发案例"不太一样——**你是 PyTorch 的"用户"而不是"开发者"**：

```
你用 MinerU 解析 PDF
  → MinerU 内部调用 PyTorch
    → PyTorch 驱动 GPU 跑 AI 模型
      → 模型做版面检测、OCR、公式识别
        → 输出 Markdown

你不需要写 PyTorch 代码，
但需要确保 PyTorch 正确装了 CUDA 版（你之前操作的那几步）。
```
