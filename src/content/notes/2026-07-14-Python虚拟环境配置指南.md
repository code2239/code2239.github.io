# Python 虚拟环境配置完全指南

> **一句话理解虚拟环境**：每个项目拥有自己独立的 Python 解释器和包仓库，互不干扰。项目 A 用 Django 4.2，项目 B 用 Django 5.0——井水不犯河水。

---

## 1. 为什么需要虚拟环境？

| 问题 | 无虚拟环境 | 有虚拟环境 |
|------|-----------|-----------|
| 依赖冲突 | 项目A需要 `torch==1.13`，项目B需要 `torch==2.0` → 只能装一个 | 各项目独立，各装各的 |
| 系统污染 | `pip install` 装到系统级，卸载不干净 | 环境删了就干净 |
| 权限问题 | 系统级安装需要 `sudo` | 用户目录下操作，无需权限 |
| 复现困难 | 不知道项目装了哪些包 | 有 `requirements.txt` / `pyproject.toml`，一键复现 |

---

## 2. 方案一：venv（Python 内置，零安装）

Python 3.3+ 自带 `venv`，**零依赖、零安装**，适合绝大多数项目。

### 2.1 创建虚拟环境

```bash
# 在项目目录下创建虚拟环境（约定命名 .venv 或 venv）
cd your_project
python3 -m venv .venv

# 执行后项目目录下会多出一个 .venv 文件夹
# .venv/
#   ├── bin/         # 激活脚本、python 解释器
#   ├── lib/         # 包安装目录
#   └── pyvenv.cfg   # 配置文件
```

### 2.2 激活虚拟环境

```bash
# macOS / Linux
source .venv/bin/activate

# Windows (Command Prompt)
.venv\Scripts\activate.bat

# Windows (PowerShell)
.venv\Scripts\Activate.ps1
```

**激活后**，终端会出现环境名提示：
```bash
(.venv) your_project $ _
```

此时 `python` 和 `pip` 都指向虚拟环境内的版本：
```bash
(.venv) $ which python
# → /path/to/your_project/.venv/bin/python

(.venv) $ which pip
# → /path/to/your_project/.venv/bin/pip
```

### 2.3 安装依赖

```bash
# 安装包
(.venv) $ pip install torch transformers datasets

# 从文件安装（最常用）
(.venv) $ pip install -r requirements.txt

# 导出当前环境（生成 requirements.txt）
(.venv) $ pip freeze > requirements.txt
```

### 2.4 退出虚拟环境

```bash
(.venv) $ deactivate
```

### 2.5 删除虚拟环境

```bash
# 直接删文件夹即可
rm -rf .venv

# 然后用同样的方式重新创建
python3 -m venv .venv
```

### 2.6 标准工作流（日常）

```bash
# 1. 克隆/创建项目
git clone xxx && cd xxx

# 2. 创建虚拟环境
python3 -m venv .venv

# 3. 激活
source .venv/bin/activate

# 4. 安装依赖
pip install -r requirements.txt

# 5. 开始开发...

# 6. 更新依赖清单（安装新包后）
pip freeze > requirements.txt

# 7. 退出
deactivate
```

### 2.7 不要提交 .venv 到 Git

```gitignore
# .gitignore 中添加
.venv/
venv/
```

**原因**：虚拟环境文件夹体积大（几十 MB 到几百 MB），且依赖操作系统架构，别人拉下来不通用。只提交 `requirements.txt` 即可。

## 3. 方案二：Conda（数据科学 / AI 标配）

Conda 不仅能管理 Python 环境，还能管理**非 Python 依赖**（如 CUDA、cuDNN、FFmpeg），是 AI/数据科学领域的事实标准。

### 3.1 安装 Miniconda（推荐）

```bash
# macOS (Apple Silicon 选 arm64, Intel 选 x86_64)
# 下载：https://docs.conda.io/en/latest/miniconda.html

# 安装后验证
conda --version
```

### 3.2 常用命令

```bash
# 创建环境，指定 Python 版本
conda create -n myenv python=3.11

# 创建环境并同时安装包
conda create -n torch_env python=3.11 pytorch torchvision -c pytorch

# 激活/退出
conda activate myenv
conda deactivate

# 列出所有环境
conda env list

# 删除环境
conda remove -n myenv --all

# 导出/导入环境
conda env export > environment.yml
conda env create -f environment.yml

# 安装包（conda VS pip 混用）
conda install numpy pandas
pip install torch transformers  # conda 没有的用 pip 补
```

### 3.3 与 venv 的关键区别

| 特性 | venv | Conda |
|------|------|-------|
| Python 版本 | 继承系统的 Python 版本 | **可指定任意版本**（3.8/3.9/3.10/3.11...） |
| 非 Python 包 | ❌ 不能 | ✅ CUDA、cuDNN、gcc、FFmpeg |
| 包数量 | pip 仓库（PyPI） | conda 仓库 + PyPI |
| 体积 | 小 | 较大 |
| AI 项目适用性 | 基本够用 | 更推荐（CUDA、GPU 版本管理方便） |

## 4. 方案三：uv（新贵，极速）

`uv` 是 Rust 编写的 Python 包管理器，**速度比 pip 快 10-100 倍**，可完全替代 pip + venv。PyCharm 2025.3+ 已将其设为默认包管理器。

```bash
# 安装 uv
curl -LsSf https://astral.sh/uv/install.sh | sh

# 创建虚拟环境
uv venv .venv

# 激活（和 venv 一样）
source .venv/bin/activate

# 安装依赖（极快！）
uv pip install -r requirements.txt
```

## 5. 实战：你的 AI 项目如何配置

基于你在学的 [[AI开发三周系统学习计划]]，推荐工作流：

### 推荐：Conda + pip 混用

```bash
# 1. 为三周学习计划创建一个统一环境
conda create -n ai_bootcamp python=3.11 -y

# 2. 激活
conda activate ai_bootcamp

# 3. 安装核心依赖
pip install httpx pydantic openai anthropic    # LLM 调用
pip install langchain chromadb                  # RAG
pip install jupyter matplotlib                  # 可视化/笔记

# 4. 验证
python -c "import httpx; import pydantic; print('✅ 环境就绪')"

# 5. 用完退出
conda deactivate
```

### 轻量：venv（每个项目独立）

```bash
# 每个 Block 创建独立环境
mkdir block_a && cd block_a
python3 -m venv .venv
source .venv/bin/activate
pip install pydantic mypy

# 切到 Block B 时
cd ../block_b
python3 -m venv .venv
source .venv/bin/activate
pip install httpx openai anthropic
```

## 6. 常见问题

### Q1: `pip` 命令走的是系统级还是环境内？
运行 `which pip` 确认路径：
- 如果在环境内 → `.venv/bin/pip` ✅
- 如果显示 `/usr/local/bin/pip` → 没有激活环境 ❌

### Q2: 为什么我的虚拟环境激活后还是调用了系统 Python？
检查 `PATH` 变量：
```bash
echo $PATH
# 应该看到 .venv/bin 在最前面：/project/.venv/bin:/usr/bin:/bin:...
```

### Q3: 删除环境后，项目还能运行吗？
**不能**。删除环境 = 删除了所有依赖包。需要重新创建并安装。

### Q4: 多人协作时怎么管理环境？
- 提交 `requirements.txt` 或 `environment.yml` 到 Git
- 团队成员各自 `pip install -r requirements.txt` 或 `conda env create -f environment.yml`
- 避免提交 `.venv/` 目录

### Q5: PyCharm/VS Code 里怎么配置？
```yaml
PyCharm:
  设置 → Project → Python Interpreter → Add Interpreter
  → 选择 "Existing environment" → 定位到 .venv/bin/python

VS Code:
  Cmd+Shift+P → "Python: Select Interpreter"
  → 选择 .venv/bin/python
  → 终端自动激活（需要在 settings.json 设置 "python.terminal.activateEnvironment": true）
```

## 7. 三方案速查表

| 场景 | 推荐方案 | 理由 |
|------|---------|------|
| 日常 Python 项目 | **venv** | 零安装、轻量、够用 |
| AI/深度学习项目 | **Conda** | CUDA 管理、Python 版本随意切换 |
| 需要极速安装 | **uv** | Rust 实现，比 pip 快 10-100x |
| 团队协作项目 | **venv + requirements.txt** | 最通用、最兼容 |
| Jupyter Notebook | **Conda** | ipykernel 注册方便 |
