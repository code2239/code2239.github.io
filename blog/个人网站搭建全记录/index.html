一、技术选型

选用技术栈

层选型理由
框架Astro 5原生 Markdown 支持、零 JS 默认输出、构建极轻
样式Tailwind CSS 4与设计 tokens 配合、按需生成
部署GitHub Pages免费、推送即部署、支持自定义域名
设计风格极简黑白纯黑白灰、无强调色，类似 Linear 风格

文件结构

xingyouhuanyu-site/
├── src/
│   ├── pages/           # 页面（.astro）
│   ├── components/      # 组件（Layout, Navigation, ArticleCard 等）
│   ├── content/blog/    # 博客文章（.md）
│   ├── lib/config.ts    # 站点配置
│   └── styles/global.css # 设计 tokens + 全局样式
├── public/             # 静态文件（favicon、自包含 HTML）
├── .github/workflows/  # GitHub Actions 自动部署
└── astro.config.mjs    # Astro 配置

二、部署流程

GitHub Pages 配置

  1. 创建仓库名为 用户名.github.io（用户主页）
  2. Settings → Pages → Source 选 “GitHub Actions”
  3. 推送代码后 Actions 自动构建部署

部署工作流

name: Deploy to GitHub Pages
on:
  push:
    branches: [main]
permissions:
  contents: read
  pages: write
  id-token: write
jobs:
  build-and-deploy:
    runs-on: ubuntu-latest
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 22
      - run: npm ci
      - run: npm run build
      - uses: actions/configure-pages@v5
      - uses: actions/upload-pages-artifact@v3
        with:
          path: './dist'
      - uses: actions/deploy-pages@v5

部署踩坑

问题原因解决
YAML 日期解析错误date: 2026-07-04 被解析为 Date 而非 string加引号：date: "2026-07-04"
gh-pages 分支部署无效用户主页（username.github.io）只支持 main 分支或 Actions用官方 actions/deploy-pages 方案
推送 gh-pages 分支 403缺少 contents: write 权限Actions 默认权限为 read-only
部署成功但网站 404Pages Source 选错（gh-pages 分支不被用户主页支持）Source 切回 “GitHub Actions”

三、内容管理

博客文章格式

每篇文章需要 YAML Frontmatter：

---
title: "文章标题"
date: "2026-07-05"
tags: ["标签1", "标签2"]
summary: "一句话摘要"
---

正文内容...

注意：日期必须加引号，否则 Ubuntu CI 上 YAML 会解析为 Date 对象导致构建失败。

上传文章流程

# 把 .md 文件放到 src/content/blog/

cd "Documents\Website\xingyouhuanyu-site"
git add .
git commit -m "新文章：xxx"
git push origin main

# 等 1-2 分钟，Actions 自动部署

上传 HTML 文件

方式放哪访问
自包含 HTML（有自己设计）public/域名/文件名.html
套网站壳、带导航转成 .astro 放 src/pages/域名/页面名

论文日报

论文日报是自包含 HTML，放在 public/papers/ 下：

public/papers/
├── papers_2026-07-02.html
├── papers_2026-07-03.html
├── papers_2026-07-04.html
└── papers_2026-07-05.html

另建 src/pages/papers.astro 作为索引页面，导航栏加”论文”链接。

四、自定义域名

购买

推荐平台：Cloudflare（成本价）、Namecheap、Porkbun。.com 年费约 60-80 元。

绑定

  1. Settings → Pages → Custom domain → 填入域名 → Save
  2. DNS 添加 CNAME 记录指向 用户名.github.io
  3. 勾选 Enforce HTTPS
