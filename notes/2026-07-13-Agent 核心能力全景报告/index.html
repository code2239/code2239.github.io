Agent 核心能力全景报告

一、概述

现代 AI Agent 不只是”聊天机器人”，而是具备思考、规划、分工、自动化、执行能力的完整智能体系统。其核心能力可归纳为 10 个层面：

┌──────────────────────────────────────┐
│          Prompt（大脑）               │
│         记忆（长期 + 短期）            │
├──────────────────────────────────────┤
│        计划模式（先想再做）            │
│        Hook 机制（自动化触发）         │
├──────────────────────────────────────┤
│        Skills（专业技能模板）          │
│        子代理（分工并行）              │
├──────────────────────────────────────┤
│  工作流编排（多代理协作扇出/汇聚）       │
│  自主循环（定时/条件驱动）             │
├──────────────────────────────────────┤
│       MCP + Tools（手脚执行）          │
└──────────────────────────────────────┘

二、十大核心能力详解

1. Prompt（系统提示词）— 战略层

Prompt 是 Agent 的”大脑”和”人格”所在，决定了思维方式和决策逻辑。

维度说明
角色定义告诉 Agent “你是谁”——比如”你是 Claudian，专注 Obsidian 知识库管理”
行为准则安全原则、工作流程、输出格式规范
领域知识内嵌的专业知识（Markdown、Git、Obsidian 生态等）
上下文理解从结构化消息中提取意图（编辑器选区、链接笔记等）

Prompt 决定了 Agent 为什么做以及怎么做。

2. Skills（技能）— 战术层

Skills 是预定义的工作流模板，相当于 Agent 的”工具箱”。每个 Skill 是一套完整的操作指南，告诉 Agent 在特定场景下一步步该怎么做。

类别典型示例
开发类code-review 代码审查 · feature-dev 功能开发 · build-fix 构建修复
设计类design-system 设计系统 · brand-guidelines 品牌指南 · frontend-design 前端设计
研究类deep-research 深度研究 · paper-lookup 论文查找 · literature-review 文献综述
运维类security-scan 安全扫描 · harness-audit 审计 · github-ops GitHub 操作
通用类dataviz 数据可视化 · update-config 配置管理

3. MCP（Model Context Protocol）— 执行层

MCP 是 Agent 与外部世界交互的标准化协议接口，核心价值在于标准化——不同 Agent 可以用同一套协议接入相同的外部工具。

能力说明
🔧 工具调用读写文件、执行命令、搜索代码
🌐 浏览器操作打开页面、截图、填写表单、运行 Lighthouse 审计
📡 外部服务连接 GitHub、数据库、文档服务等
🔌 MCP Server通过标准化协议接入第三方工具和服务

4. 子代理（Subagents）

Agent 可以派生出更专业的子代理并行工作，每个子代理拥有不同的专业角色和工具集。

主代理 (你对话的我)
  ├── 架构师子代理 → 分析系统设计
  ├── 安全审查子代理 → 检查漏洞
  ├── 性能优化子代理 → 找瓶颈
  └── 代码审查子代理 → 检查质量

优势：

  • 多个子代理并行执行，大幅提高效率
  • 每个子代理专注一个领域，输出质量更高
  • 结果汇总后由主代理做最终判断

5. 记忆与上下文管理（Memory & Context）

类型说明
短期记忆当前对话的上下文窗口，能记住前面的交流内容
长期记忆通过文件、笔记、配置文件持久化保存
会话恢复即使中断后重启，也能继续之前的工作状态
上下文压缩自动精简历史对话，保留关键信息，避免上下文溢出

6. 计划模式（Plan Mode）

在动手前先制定实施计划，让用户审批后再执行：

探索代码库 → 设计方案 → 用户确认 → 开始实施

适用场景：

  • 新功能实现
  • 代码重构
  • 影响范围较大的修改
  • 架构决策

核心价值： 避免”做错了再改”，在复杂任务中尤为重要。

7. Hook 机制（钩子）

自动触发的行为规则，不需要用户每次手动触发，完全自动化。

配置示例：

  • “每次保存代码前，自动运行 lint 检查”
  • “每次新建文件，自动添加 frontmatter 模板”
  • “每次推送代码后，自动运行测试”

Hook 配置在 settings.json 中，由系统执行而非 Agent 手动调用。

8. 自主循环（Autonomous Loops）

Agent 可以自主驱动复杂任务，不需要每一步都等用户确认：

类型说明
定时循环每隔一段时间自动执行某项任务（如每 5 分钟检查部署状态）
条件循环直到某个条件满足才停止（如”找到所有 Bug 并修复”）
预算驱动根据 token 预算动态调整执行深度
收敛循环持续迭代直到结果稳定（dry rounds 检测）

9. 工作流编排（Workflow Orchestration）

更高级的多代理协作模式，将复杂任务分解为可编排的阶段：

模式说明
流水线（Pipeline）任务依次经过多个阶段，前一阶段输出作为后一阶段输入
并行扇出（Fan-out）同一任务分给多个代理同时处理
汇聚（Converge）多个独立结果合并成一个最终输出
迭代验证生成 → 审查 → 修改 → 再审查，循环直到达标
早期退出如果某阶段结果为空，自动跳过后续阶段

10. 环境感知能力

Agent 具备对运行环境的全面感知和操作能力：

能力说明
文件系统读写文件、目录结构分析、模式匹配搜索
版本控制Git 操作、分支管理、提交和推送
网络交互网页抓取、API 调用、搜索引擎
浏览器控制页面导航、截图、填表单、运行 E2E 测试
终端执行运行命令、安装依赖、构建项目

三、三层架构总结

层级组成职责
战略层Prompt + 记忆思考方式、决策逻辑、知识沉淀
战术层Skills + 子代理 + 计划模式 + Hook标准流程、专业分工、自动化触发
执行层MCP + Tools + 工作流 + 自主循环实际操作、多代理协作、持续运行

四、一句话总结

Prompt 让我聪明，Skill 让我专业，MCP 让我能动手，Subagent 让我能分工，Hook 让我能自动化，Loop 让我能持续，Workflow 让我能协作。
