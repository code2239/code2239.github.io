图论入门：从 Obsidian 到 AI

图论是研究「连接」的数学——不关心节点是什么，只关心谁跟谁连在一起。

1. 一个直觉起点：你的 Obsidian 图谱

你在 Obsidian 里每天看到的：

一篇笔记        =  一个圆点
[[]] 链接       =  一条线
图谱全局视图    =  一张图

对应的图论术语：

Obsidian 中的概念图论术语符号定义
一篇笔记顶点 / 节点 (Vertex)$V$一个实体
一个 [[]] 链接边 (Edge)$E$两个实体之间的关系
你的整个图谱图 (Graph)$G = (V, E)$顶点和边的集合
A → B 的箭头有向边 (Directed Edge)$A \to B$关系有方向
没有箭头的线无向边 (Undirected Edge)$A - B$关系是双向的
孤立节点孤立顶点 (Isolated Vertex)—没有连接任何边的节点
笔记与笔记之间的路径路径 (Path)$v_1 \to v_2 \to … \to v_k$沿着边走，不重复
一组互相连通的笔记连通分量 (Connected Component)—内部连通，对外隔离

图论就是把你在 Obsidian 图谱里「看到」的东西，变成可以用数学精确描述和计算的东西。

2. 图的数学定义

2.1 无向图

一个无向图 G = (V, E) 包含：
  V = {v₁, v₂, v₃, ...}    ← 顶点的集合
  E = {(v₁,v₂), (v₂,v₃), ...}  ← 边的集合，边没有方向

例子：
  V = {Python笔记, AI笔记, 数据处理笔记}
  E = {(Python, AI), (AI, 数据处理)}
  → Python 和 AI 互相链接，AI 和数据处理互相链接
  → Python 和数据处理没有直接链接

2.2 有向图

每条边有方向：
  E = {(Python → AI), (AI → 数据处理)}
  → Python 链接到了 AI（但 AI 不一定链接回 Python）

在 Obsidian 中，[[]] 链接在底层是有向的（A 链接到 B ≠ B 链接到 A），但图谱展示时默认显示为双向边。

2.3 带权图

每条边有一个数值（权重）：
  Python ---[3]--- AI        ← 权重 3
  AI    ---[1]--- 数据处理     ← 权重 1

权重可以表示：
  - 链接的强度（两篇笔记被同时引用的次数）
  - 距离（两个城市之间的公里数）
  - 成本（两个节点之间的传输代价）
  - 相似度（两个向量之间的余弦相似度）

2.4 Obsidian 中的这三种图

无向图：  你在图谱中看到的默认视图
有向图：  [笔记A] 指向 [笔记B]，但B不一定指向A
带权图：  用 Graph Analysis 插件可以显示边的权重
          （基于共同引用、标签相似度等）

3. 图论核心概念（配 Obsidian 类比）

3.1 度 (Degree)

一个顶点连接的边数。

在 Obsidian 中：
  度 = 一篇笔记有多少个 [[]] 链接

  度高的笔记 = 被广泛引用的核心概念
  度低的笔记 = 冷门知识点

  Obsidian 图谱中节点的大小默认就反映度——
  越大 = 度越高 = 越核心

度的类型：

  • 度 (Degree)：无向图中的一个顶点连接的边数
  • 入度 (In-degree)：有向图中指向该顶点的边数（被谁引用）
  • 出度 (Out-degree)：有向图中从该顶点出发的边数（引用了谁）

3.2 路径 (Path)

从一个顶点到另一个顶点经过的边的序列。

在 Obsidian 中：
  找「Python基础」到「Transformer」的最短路径
  = 图谱中两个节点之间最少需要跳转几次

  路径：Python基础 → 深度学习 → Transformer
  长度：2

最短路径问题：

  • 地图导航（高德/Google Maps 的核心算法）
  • 社交网络中的「六度分隔」
  • 知识网络中两个概念之间的最短知识路径

3.3 连通分量 (Connected Component)

一个子图，内部的顶点互相连通，但与子图外的顶点没有连接。

在 Obsidian 中：
  你可能有几个知识簇：
    ├── 机器学习簇（ML笔记之间互连）
    ├── 生活笔记簇（日记、食谱全部互相连）
    └── 编程笔记簇（代码笔记之间互连）

  如果这些簇之间没有任何 [[]] 链接相连——
  你的图谱中有 3 个连通分量。

3.4 桥梁 (Bridge)

一条边，如果去掉它，图的连通分量数会增加。

在 Obsidian 中：
  你只有一篇笔记连接了「机器学习」和「编程」两个领域
  删掉这篇笔记 → 两个领域完全断开
  这篇笔记的链接就是「桥梁」

桥梁代表了你知识体系中关键的跨领域连接。

3.5 聚类系数 (Clustering Coefficient)

你的邻居之间互相连接的程度。

在 Obsidian 中：
  如果「Python笔记」链接了「NumPy笔记」和「Pandas笔记」
  而「NumPy」和「Pandas」互相也有链接
  → 聚类系数高 → 你对这个领域的理解很密实

  如果「Python笔记」链接了「NumPy笔记」和「早餐食谱」
  而「NumPy」和「早餐食谱」互不相连
  → 聚类系数低 → 这些主题只是被凑巧关联

4. 图论在 AI 中的核心应用

4.1 Transformer 的自注意力机制

这是你学 AI 时最重要的图论应用。

在一句话中：
  "The cat sat on the mat"

Transformer 把每个词看作一个节点，
每对词之间的关系看作一条带权边：

           cat
          /   \
         The   sat
          |   / |
         on    mat

每个词的表示 = 所有词对该词的「加权和」
权重 = 词与词之间的「注意力分数」

这不是比喻——Transformer 的自注意力计算的就是一个完全图 （每对顶点之间都有边），边的权重由注意力分数决定。

Token: [The, cat, sat, on, the, mat]
         ↓
自注意力矩阵（6×6）：
    The  cat  sat  on  the  mat
The  0.2  0.3  0.1  0.1  0.2  0.1    ← The 跟所有词的关系
cat  0.1  0.4  0.3  0.1  0.0  0.1
...                               ← 这就是一个带权完全图的邻接矩阵

4.2 图神经网络 (GNN)

传统神经网络（处理向量/图像）
  → 假设输入是「规整的」网格结构

图神经网络（处理图结构数据）
  → 输入可以是任意拓扑结构
  → 每个节点通过学习邻居的信息来更新自己的表示
  → 非常适合社交网络、分子结构、知识图谱

典型应用：

  • 药物发现（分子 = 原子为节点，化学键为边）
  • 推荐系统（用户和物品为节点，交互为边）
  • 知识图谱推理

4.3 知识图谱 (Knowledge Graph)

你的 Obsidian Vault 本质上就是一个个人知识图谱。

知识图谱 = 图论 + 语义

  实体：         节点
  关系：         带标签的边
  三元组：       (实体, 关系, 实体)
  例子：         (Transformer, 使用, 自注意力机制)

从 Obsidian 到知识图谱的桥接：

  • [[]] 链接 → 无标签的边（默认关系是「相关」）
  • 通过标签或元数据 → 可以给边赋予语义
  • 一个进阶玩法：用 Dataview 插件把你的笔记导出为结构化知识图谱

4.4 搜索引擎的 PageRank 算法

Google 起家的算法，本质是图论问题。

把整个互联网看作一张有向图：
  网页 = 节点
  超链接 = 有向边

PageRank 的核心直觉：
  - 被越多重要网页引用的网页越重要
  - 一个网页给出去的「票数」被均分

这跟你判断 Obsidian 中「核心笔记」的逻辑完全一样：
  - 被越多核心笔记引用的笔记越重要
  - 一篇笔记如果链接了很多内容，单条链接的价值就稀释了

5. 图论中几个著名的「图」

5.1 欧拉路径与七桥问题

图论的起源（1736 年，欧拉）

问题：哥尼斯堡有七座桥，能否不重复地走完所有桥？

欧拉的解法：
  把四块陆地抽象为 4 个节点
  把七座桥抽象为 7 条边
  证明：当且仅当奇度数顶点的个数为 0 或 2 时才有可能
  哥尼斯堡有 4 个奇度顶点 → 不可能

这是历史上第一次有人用「图」来解决问题
→ 图论诞生了

5.2 复杂网络中的「小世界」现象

六度分隔实验：
  世界上任意两个人之间，
  平均只要通过 6 个中间人就能建立联系

图论解释：
  现实世界的网络（社交、引文、互联网）
  都同时具有「高聚类系数」和「短平均路径」

你的 Obsidian 知识网络随着笔记增加，也会出现这个现象——主题内部高度互联（高聚类），不同主题之间只用几步就能到达（短路径）。

6. 从 Obsidian 中观察图论概念

实操：用你的 Vault 当图论实验室

图论概念在 Obsidian 中怎么看
度图谱中节点的大小 = 度
连通分量图谱中成簇的节点群
桥梁两个簇之间唯一的连接线
最短路径图谱中两个节点之间的最少跳数
孤立顶点没有任何连接的单个节点
稠密图 vs 稀疏图一个簇内链接密集（稠密）vs 整个图谱链接很少（稀疏）
邻接矩阵在图形视图开启「显示邻接矩阵」
中心性节点越大、位置越居中 → 中心性越高
