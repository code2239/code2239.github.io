Atlas 调研报告

生成日期：2026-07-11

说明：Atlas 并非独立软件，而是 Synthetic Sciences 公司在开源项目 OpenScience 之上构建的托管商业平台。

1. 开发者与公司背景

开发方全称

Synthetic Sciences，总部位于美国加利福尼亚州旧金山。详见 [[AI相关/AI工具/AI4S/openscience/OpenScience调研报告#1 开发者与公司背景]]。

4. 功能全景解析

主要功能

Atlas 是 OpenScience 的商业托管版——提供持久化研究图谱、托管模型、云端算力和实验追踪的一站式科研管理平台。

核心功能

4.1 🔗 持久化研究图谱（Research Graph）

Atlas 最核心的差异化功能——将科研过程组织为持续演化的图结构：

文献检索 → 假设 A → 实验 1 → 结果分析 → 论文草稿
              ↘ 假设 B → 实验 2 → 结果分析
  • 所有假设、实验、结果作为节点组织在图谱中
  • 支持跨会话上下文保持和分支探索

关键技术：图数据库 + 版本化 provenance 追踪

4.2 🧠 托管模型网关

  • 精选前沿模型，预付费钱包统一计费
  • 无需管理多个 API Key

4.3 ☁️ 云端算力

  • 托管云环境，容器化执行，20+ 计算提供商

4.4 📊 实验追踪与证据管理

功能说明
runs:record记录实验参数、代码、结果
evidence:add捕获实验证据
repo:capture锁定代码状态保证可复现

4.5 📚 文献库管理

library:add → library:search → library:ask（基于 RAG 的文献问答）

4.6 🌀 Flywheel Mode

用私有研究数据进行 SFT + RL 后训练，在特定科学任务上超越通用模型。

信息源：npm @synsci/atlas · YC Launch

5. 功能实现案例

案例：药物发现研究项目管理

场景：一个计算药物发现团队探索某激酶靶点的新抑制剂，需同时推进多条研究路线。

操作：

  1. atlas project:create "EGFR Inhibitors 2026" → 导入 20 篇文献
  2. 创建 3 个假设分支（ATP 位点/别构位点/T790M 共价抑制剂）
  3. atlas runs:record 记录每次分子对接实验
  4. atlas evidence:add 标记最有潜力的化合物
  5. atlas library:ask 基于项目库中的文献+实验结果综合分析
  6. 研究图谱可视化展示分支进展 → 团队决策

结果：每个假设、每次实验、每条证据都可追溯，团队知识无丢失。

一句话锐评

Atlas 试图成为”科研项目的操作系统”——用研究图谱把假设、实验、证据、文献编织成可追溯的知识网络。对于多分支、多人协作的计算科学团队，这种结构化组织方式比传统文件堆叠有本质提升。但作为全新产品，生态成熟度和定价合理性仍有待市场验证。
