MCP（Model Context Protocol）学习路径

📌 核心理念

MCP 是 Anthropic 推出的开放协议标准，让 AI Agent 能够以统一的方式连接外部数据源和工具。

类比理解：MCP 就像 USB-C 接口——不管是什么设备（工具/服务），插上同一个接口就能用。

第一阶段：理解概念（预计 2-3 天）

目标

搞清楚 MCP 是什么、为什么需要它、核心架构是什么。

学习内容

1.1 阅读官方文档（必须）

  • MCP 入门指南：https://modelcontextprotocol.io/introduction

    • MCP 解决了什么问题
    • 与传统 API 调用的区别
    • 协议的核心设计理念

  • 协议规范：https://modelcontextprotocol.io/specification

    • JSON-RPC 2.0 通信协议
    • 消息格式和传输方式
    • 请求/通知/响应模型

1.2 理解核心概念

概念说明类比
MCP Host使用工具的一方（如 Claude Code、Cursor）电脑
MCP Server提供工具的一方（如文件系统、数据库）USB 设备
MCP ClientHost 内部的协议客户端USB 驱动
Transport通信方式（stdio / SSE / Streamable HTTP）USB-C 线缆

1.3 理解架构

┌──────────────────────────────────────┐
│         MCP Host                      │
│  (Claude Code / Cursor / 你的应用)     │
│                                       │
│  ┌──────────────────────────────────┐ │
│  │        MCP Client                 │ │
│  │  (协议客户端，管理连接)             │ │
│  └────────┬─────────────────────────┘ │
│           │ Transport (stdio/SSE)      │
└───────────┼───────────────────────────┘
            │
┌───────────┼───────────────────────────┐
│  ┌────────▼─────────────────────────┐ │
│  │       MCP Server                  │ │
│  │  (提供 tools / resources / prompts) │ │
│  └──────────────────────────────────┘ │
│         第三方服务                       │
│  (文件系统 / 数据库 / API / ...)        │
└──────────────────────────────────────┘

✅ 阶段成果

  • 能用自己的话解释 “MCP 是什么”
  • 能画出 MCP 架构图
  • 理解 Host / Client / Server / Transport 四个角色的关系

第二阶段：使用 MCP（预计 2-3 天）

目标

学会在 Claude Code 中配置和使用 MCP Server。

学习内容

2.1 配置 MCP Server

Claude Code 中 MCP Server 的配置在 .claude/settings.json 或 .mcp.json：

{
  "mcpServers": {
    "my-server": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-filesystem", "/path/to/dir"],
      "env": {}
    }
  }
}

关键配置项：

字段说明
command启动 Server 的命令
args启动参数
env环境变量
transport传输方式（默认 stdio）

2.2 探索内置 MCP 工具

在 Claude Code 中，观察哪些能力来自 MCP：

  • 浏览器操作（Chrome DevTools MCP）
  • 文件搜索（Grep / Glob）
  • 代码阅读（Read / Write / Edit）

2.3 安装社区 MCP Server

尝试安装几个流行的 MCP Server：

# 1. 文件系统 Server
npx -y @modelcontextprotocol/server-filesystem .

# 2. GitHub Server（需要 token）
npx -y @modelcontextprotocol/server-github

# 3. PostgreSQL Server
npx -y @modelcontextprotocol/server-postgres

✅ 阶段成果

  • 能在 Claude Code 中成功配置一个 MCP Server
  • 能用 npx 启动至少 3 个不同的 MCP Server
  • 理解 MCP 配置文件的结构和各字段含义

第三阶段：开发第一个 MCP Server（预计 5-7 天）

目标

从零开发一个 MCP Server，理解协议细节。

学习内容

3.1 选择开发语言

语言SDK难度
TypeScript/Node.js@modelcontextprotocol/sdk⭐ 推荐（生态最成熟）
Pythonmcp SDK⭐ 推荐（最简洁）
Javamodelcontextprotocol/spec较复杂

3.2 Python 快速上手（推荐入门）

步骤 1：安装 SDK

pip install mcp

步骤 2：创建第一个 Server

# my_server.py
from mcp.server import Server
from mcp.types import Tool, TextContent

app = Server("my-server")

@app.tool()
def greet(name: str) -> str:
    """向用户打招呼"""
    return f"Hello, {name}!"

@app.tool()
def add(a: int, b: int) -> int:
    """计算两个数的和"""
    return a + b

@app.tool()
def search_notes(query: str) -> list[str]:
    """搜索笔记（示例）"""
    # 这里可以接入你的实际数据源
    return [f"找到关于 '{query}' 的笔记..."]

if __name__ == "__main__":
    from mcp.server.stdio import stdio_server
    import asyncio

    async def main():
        async with stdio_server() as (read, write):
            await app.run(read, write, app.create_initialization_options())

    asyncio.run(main())

步骤 3：配置到 Claude Code

{
  "mcpServers": {
    "my-tools": {
      "command": "python",
      "args": ["my_server.py"]
    }
  }
}

步骤 4：测试

在 Claude Code 中对话，让 Agent 调用你刚写的工具。

3.3 TypeScript/Node.js 版本

npm install @modelcontextprotocol/sdk
// my-server.ts
import { McpServer } from "@modelcontextprotocol/sdk/server/mcp.js";
import { StdioTransport } from "@modelcontextprotocol/sdk/server/stdio.js";

const server = new McpServer({
  name: "my-server",
  version: "1.0.0"
});

// 注册 Tool
server.tool("greet", "向用户打招呼", {
  name: { type: "string" }
}, async ({ name }) => ({
  content: [{ type: "text", text: `Hello, ${name}!` }]
}));

// 注册 Resource
server.resource("config", "config://app", async () => ({
  uri: "config://app",
  text: JSON.stringify({ theme: "dark" })
}));

// 启动
const transport = new StdioTransport();
server.connect(transport);

第四阶段：进阶开发（预计 7-10 天）

目标

掌握 MCP Server 的高级功能，能开发生产级 Server。

学习内容

4.1 Resources（资源）

from mcp.types import Resource

@app.resource("notes://{path}")
def read_note(path: str) -> str:
    """读取指定路径的笔记"""
    with open(f"vault/{path}.md", "r") as f:
        return f.read()

4.2 Prompts（提示模板）

@app.prompt("summarize")
def summarize_prompt(topic: str) -> str:
    """生成摘要的提示模板"""
    return f"请为以下主题写一篇300字的摘要：{topic}"

4.3 错误处理

from mcp.server.exceptions import McpError

@app.tool()
def delete_file(path: str) -> str:
    """删除文件"""
    if not os.path.exists(path):
        raise McpError(f"文件不存在: {path}")
    os.remove(path)
    return f"已删除: {path}"

4.4 SSE Transport（HTTP 传输）

# 从 stdio 切换到 SSE，支持远程连接
from mcp.server.sse import SseServerTransport

# 适合部署为独立服务的情况

4.5 安全最佳实践

原则说明
最小权限Server 只暴露必要的工具和资源
输入验证对所有参数做类型和范围校验
环境隔离不要暴露敏感环境变量
认证授权对外部服务调用做 token 管理
日志审计记录所有工具调用

✅ 阶段成果

  • 实现了 Resources 和 Prompts
  • 添加了完善的错误处理
  • 理解 SSE 和 stdio 两种传输方式的区别
  • 能按照安全最佳实践编写 Server

常见问题

Q: 需要先学什么语言？

A: Python（最快上手）或 TypeScript（生态最成熟），二选一即可。

Q: MCP 和普通 API 有什么区别？

A: MCP 是面向 AI Agent 设计的协议，标准化工具描述、资源访问和提示模板，让 Agent 能自动发现和调用工具。

Q: 需要先理解 JSON-RPC 吗？

A: 了解即可，SDK 封装了大部分细节，不需要深入 JSON-RPC 底层。

Q: 如何调试 MCP Server？

A: 使用官方 MCP Inspector 工具：

npx @modelcontextprotocol/inspector python my_server.py
